## نظرة عامة

بعد التعديل، النظام يدعم **حجز المواعيد حسب ساعات العمل**:

- **الحجز (أونلاين أو عيادة) غير متاح افتراضياً** لأي يوم حتى يضيف الأدمن **ساعات عمل** لذلك اليوم.
- عند تعيين **تاريخ + وقت البداية + وقت النهاية** يصبح اليوم قابلاً للحجز.
- كل ساعة عمل تُقسّم إلى **مواعيد (Slots)** بساعة واحدة، وكل موعد يستوعب **حد أقصى 10 حجوزات**.
- لا يمكن الحجز في موعد انتهى وقته (في الماضي)، ولا حجز مزدوج لنفس المريض في نفس الموعد.

### الميزات الرئيسية

| الميزة | الوصف |
|--------|--------|
| تعطيل الحجز افتراضياً | لا حجز حتى يحدد الأدمن ساعات العمل للتاريخ |
| مواعيد بساعة واحدة | من بداية العمل إلى النهاية، كل ساعة = موعد واحد |
| حد 10 حجوزات لكل موعد | عند اكتمال العشرة يصبح الموعد غير متاح |
| منع المواعيد الماضية | لا يمكن اختيار موعد انتهى وقته |
| منع الحجز المزدوج | نفس المريض لا يحجز نفس اليوم ونفس الموعد مرتين |
| رسائل عربية واضحة | لكل حالة فشل رسالة محددة بالعربية |

---

## قواعد ساعات العمل

1. **بدون يوم عمل (Working Day)** للتاريخ المطلوب → الحجز غير متاح، والرسالة: **الحجز غير متاح اليوم**.
2. **بدون وقت بداية/نهاية** من الأدمن → نفس الرسالة.
3. **الموعد ممتلئ** (10 حجوزات في نفس الساعة) → **هذا الموعد ممتلئ**.
4. **كل المواعيد ممتلئة في ذلك اليوم** → **مواعيد اليوم اكتملت**.
5. **كل المواعيد المتبقية في الماضي** → **انتهت مواعيد الحجز لليوم**.

---

## الرسائل بالعربية

جميع حالات الفشل تُرجع رسالة واضحة بالعربية:

| الرسالة | متى تُستخدم |
|---------|-------------|
| **الحجز غير متاح اليوم** | لا يوجد يوم عمل لهذا التاريخ، أو الموعد خارج ساعات العمل |
| **هذا الموعد ممتلئ** | وصل عدد الحجوزات في هذا الموعد إلى 10 |
| **مواعيد اليوم اكتملت** | كل المواعيد في اليوم إما ممتلئة أو في الماضي |
| **انتهت مواعيد الحجز لليوم** | كل المواعيد المتبقية أصبحت في الماضي (لا يوجد وقت متاح لاحقاً) |

## نماذج قاعدة البيانات

### WorkingDay (يوم العمل)

| الحقل | النوع | الوصف |
|-------|-------|--------|
| `id` | INTEGER | المعرّف |
| `date` | DATEONLY | التاريخ (YYYY-MM-DD) — فريد |
| `startTime` | STRING(5) | وقت البداية مثل 10:00 |
| `endTime` | STRING(5) | وقت النهاية مثل 18:00 |
| `isActive` | BOOLEAN | مفعّل أم لا |
| `createdBy` | INTEGER | معرّف الأدمن الذي أنشأ السجل |

### Patient (المريض)

| الحقل | النوع | الوصف |
|-------|-------|--------|
| `id` | INTEGER | المعرّف |
| `name` | STRING | الاسم |
| `phone` | STRING | رقم الهاتف |
| `email` | STRING | البريد (اختياري) |

### Booking — الحقول الجديدة للحجز بالمواعيد

| الحقل | النوع | الوصف |
|-------|-------|--------|
| `patientId` | INTEGER | معرّف المريض (لحجوزات الـ slot) |
| `slotDate` | DATEONLY | تاريخ الموعد YYYY-MM-DD |
| `timeSlot` | STRING(5) | وقت الموعد مثل 10:00 |
| `bookingType` | ENUM | `online` \| `clinic` |
| `status` | ENUM | `confirmed` \| `cancelled` (وغيره للحجوزات القديمة) |

- **فهارس**: `(slotDate, timeSlot)` و `(patientId, slotDate, timeSlot)` لتحسين الأداء والعدّ.

## واجهات الحجز بالمواعيد

### 1. الحصول على المواعيد المتاحة

يعيد المواعيد المتاحة لتاريخ معيّن، أو رسالة عربية عند عدم الإتاحة.

- **المسار:** `GET /api/bookings/available-slots`
- **الصلاحية:** عام (بدون توكن)
- **Query:**

| المعامل | النوع | مطلوب | الوصف |
|---------|--------|--------|--------|
| `date` | string | نعم | التاريخ بصيغة YYYY-MM-DD |

**مثال طلب:**
```http
GET /api/bookings/available-slots?date=2026-02-20
```

**استجابة ناجحة (200):**
```json
{
  "available": true,
  "date": "2026-02-20",
  "workingHours": { "start": "10:00", "end": "18:00" },
  "slots": [
    { "timeSlot": "10:00", "count": 3, "available": true },
    { "timeSlot": "11:00", "count": 10, "available": false },
    { "timeSlot": "12:00", "count": 0, "available": true }
  ],
  "availableSlots": ["10:00", "12:00", "13:00", "14:00", "15:00", "16:00", "17:00"]
}
```

**عند عدم الإتاحة (400):**
```json
{
  "available": false,
  "message": "الحجز غير متاح اليوم"
}
```

أو: `مواعيد اليوم اكتملت` أو `انتهت مواعيد الحجز لليوم` حسب الحالة.

---

### 2. إنشاء حجز موعد (أونلاين أو عيادة)

- **المسار:** `POST /api/bookings/slots`
- **الصلاحية:** عام
- **Body (JSON):**

| الحقل | النوع | مطلوب | الوصف |
|-------|--------|--------|--------|
| `patientId` | number | نعم | معرّف المريض (من واجهة المرضى) |
| `date` | string | نعم | YYYY-MM-DD |
| `timeSlot` | string | نعم | مثل 10:00 |
| `bookingType` | string | لا | `online` (افتراضي) أو `clinic` |

**مثال طلب:**
```http
POST /api/bookings/slots
Content-Type: application/json

{
  "patientId": 1,
  "date": "2026-02-20",
  "timeSlot": "10:00",
  "bookingType": "online"
}
```

**استجابة ناجحة (201):**
```json
{
  "message": "تم الحجز بنجاح",
  "booking": {
    "id": 5,
    "patientId": 1,
    "slotDate": "2026-02-20",
    "timeSlot": "10:00",
    "bookingType": "online",
    "status": "confirmed",
    "customerName": "أحمد محمد",
    "customerPhone": "01012345678",
    "createdAt": "..."
  }
}
```

**أمثلة أخطاء (400):**
- `"message": "الحجز غير متاح اليوم"`
- `"message": "هذا الموعد ممتلئ"`
- `"message": "انتهت مواعيد الحجز لليوم"`
- `"message": "لديك حجز بالفعل في هذا الموعد"`
- `"message": "المريض غير موجود"`

---

### 3. إلغاء حجز موعد

- **المسار:** `PATCH /api/bookings/slots/:id/cancel`
- **الصلاحية:** عام
- **المعامل:** `id` = معرّف الحجز

**مثال طلب:**
```http
PATCH /api/bookings/slots/5/cancel
```

**استجابة ناجحة (200):**
```json
{
  "message": "تم إلغاء الحجز",
  "booking": { "id": 5, "status": "cancelled", ... }
}
```

## واجهات ساعات العمل

جميعها تتطلب **تسجيل دخول أدمن** مع الهيدر:
```http
Authorization: Bearer <token>
```

### 1. تعيين ساعات عمل ليوم معيّن

- **المسار:** `POST /api/admin/working-days`
- **Body:**
```json
{
  "date": "2026-02-20",
  "startTime": "10:00",
  "endTime": "18:00"
}
```

**استجابة (201):**
```json
{
  "message": "تم تعيين ساعات العمل بنجاح",
  "workingDay": {
    "id": 1,
    "date": "2026-02-20",
    "startTime": "10:00",
    "endTime": "18:00",
    "isActive": true,
    "createdBy": 1
  }
}
```

- إذا وُجد يوم عمل لنفس التاريخ يُحدَّث (upsert).

---

### 2. تحديث يوم عمل

- **المسار:** `PUT /api/admin/working-days/:id`
- **Body (اختياري):** `date`, `startTime`, `endTime`, `isActive`

---

### 3. استعلام أيام العمل

- **المسار:** `GET /api/admin/working-days`
- **Query:**
  - `date=YYYY-MM-DD` → يرجع يوم العمل لهذا التاريخ فقط (أو 404 مع رسالة "الحجز غير متاح اليوم").
  - `startDate` و `endDate` → قائمة أيام العمل في المدى.

**أمثلة:**
```http
GET /api/admin/working-days?date=2026-02-20
GET /api/admin/working-days?startDate=2026-02-01&endDate=2026-02-28
```

## واجهات المرضى

يُستخدم المريض للحجز بالمواعيد: إنشاء/البحث بالهاتف ثم استخدام `patient.id` في `POST /api/bookings/slots`.

### إنشاء أو البحث عن مريض بالهاتف

- **المسار:** `POST /api/patients`
- **الصلاحية:** عام
- **Body:**
```json
{
  "name": "أحمد محمد",
  "phone": "01012345678",
  "email": "ahmed@example.com"
}
```
- `name` و `email` اختياريان. إذا وُجد مريض بنفس `phone` يُحدَّث ويُرجع، وإلا يُنشأ مريض جديد.

**استجابة (200):**
```json
{
  "patient": {
    "id": 1,
    "name": "أحمد محمد",
    "phone": "01012345678",
    "email": "ahmed@example.com"
  }
}
```

## سير العمل الكامل

1. **الأدمن:** تعيين ساعات العمل للتاريخ.
   ```http
   POST /api/admin/working-days
   { "date": "2026-02-20", "startTime": "10:00", "endTime": "18:00" }
   ```

2. **العميل/التطبيق:** جلب المواعيد المتاحة.
   ```http
   GET /api/bookings/available-slots?date=2026-02-20
   ```

3. **العميل:** إنشاء أو جلب المريض.
   ```http
   POST /api/patients
   { "name": "أحمد محمد", "phone": "01012345678" }
   ```

4. **العميل:** إنشاء الحجز.
   ```http
   POST /api/bookings/slots
   { "patientId": 1, "date": "2026-02-20", "timeSlot": "10:00", "bookingType": "online" }
   ```

5. **إلغاء عند الحاجة:**
   ```http
   PATCH /api/bookings/slots/5/cancel
   ```

## التحديث الفوري (Real-time)

عند تغيير **حالة الكشف** (في الانتظار / تم الكشف) من الأدمن، يتم بث التحديث فوراً لكل من يشاهد حجوزات نفس اليوم **بدون الحاجة لعمولة ريفرش**.

### آلية العمل

- السيرفر يستخدم **Socket.io** على نفس البورت (مثلاً `http://localhost:8000`).
- الواجهة الأمامية تتصل بـ Socket ثم **تشترك في تاريخ معيّن** (مثلاً `2026-02-24`).
- عند استدعاء `PATCH /api/bookings/:id/examination-status` يبث السيرفر حدث `bookings:updated` لكل العملاء المشتركين في ذلك التاريخ.

### استخدام الواجهة الأمامية (Frontend)

1. تثبيت العميل: `npm install socket.io-client`
2. عند فتح صفحة حجوزات يوم معيّن (مثلاً `date=2026-02-24`):
   - الاتصال: `const socket = io('http://localhost:8000')`
   - الاشتراك في اليوم: `socket.emit('bookings:subscribe', '2026-02-24')`
   - الاستماع للتحديثات: `socket.on('bookings:updated', (payload) => { ... })`
3. عند استلام `bookings:updated`: تحديث الحجز في القائمة (مثلاً تحديث `examinationStatus` للحجز `payload.booking.id`) أو إعادة جلب القائمة من `GET /api/bookings/all?date=2026-02-24`.
4. عند مغادرة الصفحة أو تغيير التاريخ: `socket.emit('bookings:unsubscribe', '2026-02-24')`

### شكل الـ payload

```json
{
  "action": "examinationStatus",
  "booking": {
    "id": 5,
    "examinationStatus": "done",
    "customerName": "...",
    ...
  }
}
```

## الحجوزات القديمة

النظام ما زال يدعم الحجوزات **بدون مواعيد (Slots)** كما كانت:

- **حجز أونلاين (بدون تاريخ موعد):** `POST /api/bookings/online` — الاسم، الهاتف، العمر، نوع الكشف.
- **حجز عيادة (مع تاريخ):** `POST /api/bookings/clinic` — يتطلب صلاحية `manage_daily_bookings`.
- **عرض وإدارة الحجوزات:** `GET /api/bookings/all`, `GET /api/bookings/online`, تحديث الحالة، إلغاء، تقرير المريض، حالة الكشف — كلها كما في التوثيق السابق (`BOOKING_MANAGEMENT_API.md`).

الحجوزات الجديدة بالمواعيد تستخدم `slotDate` و `timeSlot` و `patientId` وتخضع لقواعد ساعات العمل والحد الأقصى 10 لكل موعد والرسائل العربية أعلاه.

## ملخص المسارات الجديدة

| Method | المسار | الوصف |
|--------|--------|--------|
| GET | `/api/bookings/available-slots?date=YYYY-MM-DD` | المواعيد المتاحة أو رسالة عربية |
| POST | `/api/bookings/slots` | إنشاء حجز موعد (أونلاين/عيادة) |
| PATCH | `/api/bookings/slots/:id/cancel` | إلغاء حجز موعد |
| POST | `/api/admin/working-days` | تعيين ساعات عمل (أدمن) |
| PUT | `/api/admin/working-days/:id` | تحديث يوم عمل (أدمن) |
| GET | `/api/admin/working-days?date=...` أو `?startDate=&endDate=` | استعلام أيام العمل |
| POST | `/api/patients` | إنشاء/البحث عن مريض بالهاتف |
